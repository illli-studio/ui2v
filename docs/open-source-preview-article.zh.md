# 开源渲染器预览

[English](open-source-preview-article.md)

ui2v 探索的是一种结构化视频工作流：用 JSON 和代码描述界面动效，用确定性的
运行时计算状态，在真实浏览器里渲染，最后导出为视频文件。

## 为什么是结构化视频

很多 UI 视频并不是摄像机素材，而是界面动效、数据叙事、产品演示、图解、
动态排版或生成式视觉系统。相比手动编辑视频轨道，这类内容往往更适合用结构化
场景和时间线描述。

结构化项目有几个优势：

- 文本、颜色、时间、布局和数据可以直接编辑。
- 同一个模板可以生成很多版本。
- AI 系统可以生成可检查的计划，而不是不透明像素。
- 渲染结果可以进入 CI 或自动化内容管线验证。

## 这个仓库开放了什么

这个仓库聚焦于渲染基础：

- JSON 项目模型。
- 用于场景图和时间线计算的 Runtime Core。
- 用于视觉输出的浏览器 Canvas 引擎。
- 用于预览和 MP4 导出的 Puppeteer producer。
- 用于校验、检查、预览和渲染的 CLI。

它不是完整可视化编辑器，而是其他编辑器、agent、脚本和自动化系统可以构建在
其上的底层渲染器。

## 基于浏览器的渲染

渲染器有意使用浏览器处理浏览器原生任务。Canvas、兼容 DOM 的库、WebGL
生态和 WebCodecs 都已经在浏览器中存在。CLI 使用 Puppeteer 启动这个环境，
再由 Node.js 写出最终文件。

主路径避开 Electron、FFmpeg 和原生 canvas 绑定。

## 当前边界

- MP4 是主要生产输出格式。
- AVC/H.264 是默认编码器。
- 浏览器依赖可能通过固定 CDN URL 加载。
- 编码结果目前会先以 base64 返回 Node，再写入文件。

这些边界会明确写出，方便贡献者和集成方在渲染器继续成熟时规划使用方式。

## 方向

长期目标是为生成式、可编程、可重复的 UI 视频提供可靠渲染器。后续方向包括
更强的离线依赖处理、流式输出、更多适配器、更丰富示例和更好的检查工具。
